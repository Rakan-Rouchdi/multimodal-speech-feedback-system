# Implementation Challenges and Solutions

| Challenge | Why it mattered | Implemented solution | Evidence | Remaining limitation |
|---|---|---|---|---|
| CrisperWhisper comma-heavy transcripts | Raw transcripts used commas as word separators, which affected text metrics and readability. | `clean_transcript()` replaces commas with spaces and normalises whitespace. | `app/transcription/crisper_whisper.py`, `tests/test_crisper_whisper.py` | Cleaning is rule-based and may not handle all ASR formatting patterns. |
| Disfluency preservation | Removing `[UH]`/`[UM]` would hide important speech behaviour. | Bracket markers are converted into words (`uh`, `um`) for cleaned text, while raw transcript remains stored. | `clean_transcript`, `count_disfluencies`, `count_fillers` | Only configured disfluency markers are counted. |
| Incomplete long transcriptions | VAD could cut off longer files, causing truncated text metrics. | Long WAV files with fewer than 20 words are retried with `vad_filter=False`. | `CrisperWhisperTranscriber._looks_incomplete`, `test_crisper_whisper_retries_without_vad_when_long_audio_looks_incomplete` | Heuristic threshold may not catch every truncation. |
| Missing word timestamps in cache | Cached segment-only entries would not support downstream timestamp evidence. | Cache validation requires word fields `text`, `start_s`, `end_s`. | `app/transcription/cache.py::_entry_is_valid`, `test_transcription_cache_rejects_entries_without_word_timestamps` | Cache is JSON and lacks locking. |
| Transcription latency | CrisperWhisper can be slow over many files. | Optional cache for single runs; enabled by default in batch evaluation. | `transcribe_with_cache`, `run_batch.py` | First run still requires model transcription. |
| Missing modality handling | `speech_only` and `text_only` should not crash or get unfair perfect scores. | `available` flags and `coverage_adjusted_score()` shrink partial evidence toward neutral. | `app/pipeline/runner.py`, `app/scoring/scoring.py` | Neutral shrink is heuristic. |
| Silent/short audio | Acoustic functions can fail on empty or tiny waveforms. | Guards return zero metrics; tests cover silent and very short audio. | `app/speech_analysis/metrics.py`, `tests/test_speech_metrics.py` | Zero metrics are robust but not semantically rich. |
| ASR punctuation unreliability | Missing full stops created artificial long sentences. | `clause_lengths()` splits very long chunks into 20-word ASR-safe windows. | `app/text_analysis/metrics.py`, `test_avg_clause_length_is_not_broken_by_missing_full_stops` | Window size is heuristic. |
| Scoring threshold alignment | Initial formulas did not align well with human scores. | Weights and thresholds were refined using `outputs/main_eval/human_scores.csv`; evaluation script reports Spearman/Pearson/MAE. | `app/scoring/scoring.py`, `scripts/evaluate_against_humans.py`, `outputs/main_eval/human_evaluation_overall.csv` | In-sample refinement; needs held-out validation for generalisation. |
| Emotion model overhead | Loading TensorFlow/Keras can slow or complicate normal runs. | Emotion model is lazy-loaded and optional. | `app/emotion/predictor.py`, `run_pipeline(..., use_emotion=False)` default | Full model inference not covered by default tests. |
